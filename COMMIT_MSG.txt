test(ci): Testcontainers 제거하고 GitHub Actions Service Container 사용

Testcontainers 의존성을 제거하고 GitHub Actions의 Service Container를 사용하도록 변경했습니다.

What:
- build.gradle.kts에서 Testcontainers 의존성 제거
- 통합 테스트 파일들에서 @Testcontainers, @Container, @DynamicPropertySource 제거
- GitHub Actions 워크플로우에 MongoDB와 Redis 서비스 컨테이너 추가
- 테스트 설정 파일에 고정된 DB 주소 설정

Why:
- DockerClientProviderStrategy 에러로 인한 테스트 실패 해결
- CI 환경에서 Docker 없이도 테스트 실행 가능하도록 안정성 확보
- 로컬과 CI 환경의 테스트 인프라 일관성 유지
