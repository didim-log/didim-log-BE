docs(api): API 명세서 최신화 및 클린코드 리팩토링

API 명세서를 실제 코드베이스와 동기화하고, 클린코드 원칙 위반 사항을 수정했습니다.

What:
- API_SPECIFICATION.md에 누락된 DELETE 엔드포인트 추가 (DELETE /retrospectives/{id})
- API_SPECIFICATION.md의 페이지네이션 예시를 1-based로 수정 (page=0 -> page=1)
- else 키워드를 Early Return 패턴으로 변경 (RetrospectiveController, RetrospectiveRepositoryImpl, ProblemCollectorService)
- 코드 가독성 개선 및 클린코드 원칙 준수

Why:
- API 명세서와 실제 구현 간 불일치 해결
- 우아한 테크코스 클린코드 원칙 준수 (else 예약어 사용 금지)
- 코드 유지보수성 및 가독성 향상
