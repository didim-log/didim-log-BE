# 회고 작성 표준 가이드라인 (Retrospective Standards)

이 문서는 사용자와 AI가 회고를 작성할 때 따라야 할 **불변의 목차 구조**를 정의합니다.

> 정책: DidimLog의 AI는 **로그(Log) 한 줄 평가** 용도로만 사용합니다.  
> 회고(Retrospective) 템플릿은 **사용자가 직접 작성할 목차**와 **본인 코드 블록**을 포함한 마크다운을 제공합니다.

---

## 키워드 자동 채움 정책 (Static Template)

정적 회고 템플릿의 `학습 키워드`는 **문제 메타데이터 기반으로 일부 자동 채움**됩니다.

- **자동 채움**: 문제 카테고리(한글), 난이도(`{Tier} {level}`), 문제 태그(tags)
- **사용자 편집**: 자동 채움된 키워드는 회고 작성 시 자유롭게 수정/추가/삭제 가능합니다.

## 1. 성공 회고 (Success Log)

> 문제를 맞혔을 때, 더 나은 코드를 위해 작성하는 양식입니다.

### [User 작성 영역]

1. **접근 방법 (Approach)**

   - 문제를 해결하기 위해 어떤 알고리즘이나 자료구조를 선택했나요?

   - 풀이의 핵심 로직을 한 줄로 요약해 보세요.

2. **복잡도 분석 (Complexity)**

   - 시간 복잡도: O(?)

   - 공간 복잡도: O(?)

3. **리팩토링 포인트 (Refactoring)**

   - 개선할 수 있는 변수/함수명, 중복 제거, 로직 단순화 포인트를 적어보세요.

4. **다른 풀이와 비교 (Comparison)**

   - 다른 사람의 풀이(또는 표준 풀이)와 비교해서 내 풀이의 장단점을 정리해보세요.

5. **다음 액션 (Next)**

   - 다음에 같은 유형을 만나면 어떤 점을 더 잘할지 한 줄로 적어보세요.

### 정적 템플릿 출력 예시 (Success - Static Template)

```markdown
# 🏆 [백준/BOJ] {problemId}번 {problemTitle} ({language}) 해결 회고

## 🔑 학습 키워드

- {problem.category.koreanName} *(자동 채움)*
- {problem.difficulty.name} {problem.level} *(자동 채움)*
- (추가로 떠오르는 키워드를 더 적어보세요)

## 1. 접근 방법 (Approach)

- 풀이의 핵심 로직을 한 줄로 요약해 보세요.

## 2. 복잡도 분석 (Complexity)

- 시간 복잡도: O(?)
- 공간 복잡도: O(?)

## 3. 리팩토링 포인트 (Refactoring)

- 개선할 수 있는 변수/함수명, 중복 제거, 로직 단순화 포인트를 적어보세요.

## 4. 다른 풀이와 비교 (Comparison)

## 5. 다음 액션 (Next)

## 2. 실패 회고 (Failure Log)

> 오답이거나 풀지 못했을 때, 원인을 분석하고 배우기 위한 양식입니다.

### [User 작성 영역]

1. **실패 현상 (Symptom)**

   - 어떤 종류의 에러가 발생했나요? (시간 초과, 메모리 초과, 틀렸습니다, 런타임 에러)

   - 테스트 케이스 중 통과하지 못한 예시가 있나요?

2. **나의 접근 (My Attempt)**

   - 어떤 로직으로 풀려고 시도했나요?

3. **원인 추정 (Root Cause)**

   - 왜 실패했다고 생각하나요? (논리/구현/복잡도/입출력 등)

4. **반례/재현 케이스 (Counter Example)**

   - 내 코드를 무너뜨리는 치명적인 입력값(Edge Case)은 무엇인가요?

5. **다음 시도 계획 (Next)**

   - 다음 시도에서 바꿀 점을 체크리스트로 적어보세요.
